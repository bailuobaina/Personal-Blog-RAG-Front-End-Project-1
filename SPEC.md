# 个人主页前端项目详细规划文档

## 一、项目概述

### 1.1 项目背景

本项目是一个基于RAG（Retrieval-Augmented Generation，检索增强生成）技术的个人主页前端应用。通过结合向量检索与AI对话能力，为访问者提供智能化的交互式个人展示体验。项目采用Vue 3作为前端框架，配合现代化的UI设计，打造一个高性能、交互流畅的个人主页平台。

### 1.2 核心特性

本个人主页系统具备以下核心功能：个人简介与作品集展示、技能与经验展示、RAG智能问答系统、项目案例浏览、联系方式展示、暗黑模式支持、响应式设计。RAG技术的引入使得个人主页不仅仅是静态的信息展示，更是一个可交互的AI助手，访问者可以针对你的背景、技能、项目等进行智能提问，系统会基于知识库生成个性化的回答。

### 1.3 目标用户

本项目主要面向以下用户群体：技术求职者、需要展示个人品牌的开发者、自由职业者、创业者、需要建立专业形象的任何个人。对于技术求职者而言，这是一个展示技术能力、让面试官通过AI问答深入了解自己的创新方式；对于自由职业者而言，这是一个让潜在客户快速了解自己能力和项目经验的智能门户。

### 1.4 与传统个人博客的区别

本项目定位为**个人主页**而非多人博客系统，主要区别在于：内容完全由个人维护，无需用户系统和权限管理；核心功能是个人品牌展示而非内容发布；RAG问答系统服务于个人介绍，而非通用知识问答；结构更精简，聚焦于个人核心信息展示。

## 二、技术栈选型

### 2.1 核心框架

前端核心框架选择Vue 3，这是Vue.js的最新版本，采用了Composition API作为核心编程范式，提供了更好的TypeScript支持、更小的包体积以及更优秀的性能。Vue 3的响应式系统经过重写，基于Proxy实现，相比Vue 2的Object.defineProperty方案，在响应速度和内存占用方面都有显著提升。Composition API允许开发者将逻辑组织为可组合的函数，便于代码复用和模块化开发。

### 2.2 构建工具

项目构建采用Vite作为构建工具。Vite是Vue团队打造的新一代前端构建工具，利用浏览器原生ES模块实现即时的开发服务器启动和热模块替换（HMR）。在生产环境下，Vite使用Rollup进行优化打包，能够生成高度优化的静态资源。相比Webpack，Vite具有更快的启动速度、更高效的热更新体验，以及更简洁的配置方式。开发环境下无需打包操作，依赖通过esbuild进行预构建，实现毫秒级的模块加载。

### 2.3 状态管理

状态管理选用Pinia，这是Vue官方推荐的新一代状态管理库。Pinia相比Vuex具有更简洁的API设计、更好的TypeScript支持、更轻量的包体积（仅约1KB），并且消除了Vuex中的mutation概念，使状态变更加可预测。Pinia采用模块化设计，每个store可以独立定义，支持组合式API风格的定义方式，便于与组件逻辑进行整合。

### 2.4 路由管理

路由管理采用Vue Router 4，这是Vue 3的官方路由库。Vue Router 4保持了与Vue 2版本类似的API设计，同时针对Vue 3进行了优化，支持Composition API、Suspense等新特性。路由守卫机制得以保留，支持全局守卫、路由独享守卫和组件内守卫，用于实现权限控制、页面标题动态修改、滚动位置恢复等功能。

### 2.5 UI框架

UI框架选择Element Plus，这是Element UI的Vue 3版本。Element Plus提供了丰富的组件库，包括按钮、表单、卡片、对话框、导航菜单等常用组件，能够快速构建现代化的界面。作为目前最成熟的Vue 3组件库之一，Element Plus拥有完善的文档、活跃的社区支持以及良好的可访问性支持。

### 2.6 HTTP客户端

网络请求采用Axios，这是一个基于Promise的HTTP客户端，可在浏览器和Node.js环境中使用。Axios提供了简洁的API接口，支持请求拦截器、响应拦截器、自动转换JSON数据、请求取消、错误处理等功能。通过封装统一的请求实例，可以实现请求 loading状态管理、Token自动注入、错误统一处理等特性。

### 2.7 RAG集成

RAG功能的实现需要与后端API进行交互，前端主要负责：用户提问界面的构建、问题提交与答案展示、对话历史的维护、引用来源的展示。在UI层面，需要设计友好的问答对话框，支持Markdown渲染、代码高亮、复制功能等。

### 3.2 目录设计原则

项目目录设计遵循以下原则：首先是功能模块化，将相关功能的代码组织在同一个目录下，便于查找和维护；其次是分层设计，从api到stores再到views，形成清晰的数据流向；第三是类型安全，在types目录下集中管理TypeScript类型定义，确保全项目类型一致性；第四是组合优先，推荐使用composables封装可复用的逻辑状态，实现逻辑的复用。

## 四、页面结构设计

### 4.1 页面概览

本项目包含以下核心页面：首页（Home）展示个人核心信息，包括简介、技能、项目精选；项目展示页（Projects）详细展示所有项目案例；经历时间线页（Timeline）展示教育背景、工作经历、成长历程；联系方式页（Contact）展示联系方式和社交链接。首页同时内嵌RAG智能问答组件，支持全站随时提问。

### 4.2 首页设计

首页是用户访问网站的入口，需要在首屏展示最重要的个人信息。页面结构分为以下几个区域：顶部固定导航栏，包含网站Logo、导航菜单和主题切换按钮；首屏区域展示个人头像、姓名、职业title和一句话简介；关于简介区域用简洁的语言介绍个人背景和核心能力；技能展示区域以标签云或分类形式展示技术栈；项目精选区域展示2-3个代表性项目卡片；页脚包含社交链接和版权信息。

首页设计需要考虑视觉效果，使用淡入动画增强体验感。首屏加载时间应控制在1.5秒以内，图片采用懒加载策略。整体风格应简洁大气，突出个人品牌。

### 4.3 项目展示页设计

项目展示页用于详细展示所有项目案例，是个人能力的重要证明。页面布局分为以下几个部分：顶部页面标题和简短描述；项目列表区域，每个项目包含项目名称、简介、技术栈、链接（GitHub/在线演示）；项目详情弹窗，点击项目卡片可查看更详细的信息，包括项目背景、功能特点、技术实现难点、个人收获等。

项目卡片设计应突出项目亮点，使用高质量的截图或封面图。对于开源项目提供GitHub链接，对于商业项目可以提供演示链接或脱敏后的描述。

### 4.4 经历时间线页设计

经历时间线页按时间顺序展示个人的教育背景和工作经历。页面设计采用垂直时间线布局，左侧或中间显示时间节点，右侧或两侧显示具体事件。每个事件包含时间范围、事件标题（如学校名称、公司名称）、事件描述（如职位、成就）。事件可以按时间正序或倒序排列。

时间线设计应清晰易读，不同类型的事件（如教育、工作）可以使用不同的图标或颜色区分。可以添加里程碑式的关键成就高亮显示。

### 4.5 联系方式页设计

联系方式页集中展示所有联系方式和社交链接。页面内容包括：邮箱地址（可点击直接打开邮件客户端）、GitHub主页链接、掘金/知乎等技术社区链接、个人微信/QQ（可选）、地理位置（城市即可，无需详细地址）。还可以添加简短的打赏二维码（如需要）。

社交链接应使用统一的图标风格，点击后在新窗口打开。页面底部可以添加留言表单，访问者可以发送留言到指定邮箱。

### 4.6 响应式布局

所有页面采用响应式设计，适配以下屏幕尺寸：超小屏幕（xs）小于576px，主要针对手机竖屏；小屏幕（sm）576px至768px，针对手机横屏；中等屏幕（md）768px至992px，针对平板竖屏；大屏幕（lg）992px至1200px，针对平板横屏；超大屏幕（xl）大于1200px，针对桌面显示器。

响应式实现采用CSS媒体查询结合Element Plus的响应式栅格系统。移动端采用底部导航栏替代顶部导航，项目列表改为单列布局。交互设计需要考虑触摸操作，按钮尺寸不小于44px点击区域。

## 五、组件设计

### 5.1 通用组件

通用组件是项目中复用性最高的组件，被各个页面广泛使用。以下是主要通用组件的设计说明：

AppHeader组件是网站的顶部导航栏，包含个人Logo或姓名缩写、导航链接集合、主题切换按钮。导航链接通过路由配置动态生成，支持高亮当前路由。主题切换支持亮色和暗黑两种模式，切换动画平滑过渡，主题偏好保存在本地存储。

AppFooter组件是网站的底部版权区域，包含姓名、社交链接、版权声明。社交链接支持GitHub、掘金、知乎、邮箱等常见平台。链接图标使用Icon组件统一管理。

Loading组件提供多种加载状态展示，包括全屏加载指示器、局部加载骨架屏。骨架屏用于内容加载占位，提升加载过程的用户体验。

### 5.2 个人展示组件

Avatar组件是个人头像展示组件，支持头像图片展示、hover效果、可选的在线状态指示器。头像设计采用圆形或圆角方形，配合简洁的边框效果。

SkillTag组件是技能标签组件，以标签形式展示各项技能。支持不同类型技能使用不同颜色区分（如前端技能、后端技能、工具技能等），hover状态下显示技能相关的简要说明。

Timeline组件是时间线组件，用于展示经历事件。组件接收事件数组作为props，支持自定义时间线样式、图标、连接线样式等。

### 5.3 项目组件

ProjectCard组件是项目卡片组件，用于列表展示。组件接收项目对象作为props，包括项目名称、封面图、简介、技术栈、链接等信息。卡片设计采用图片+内容的垂直布局，封面图使用懒加载优化性能。hover状态下显示操作按钮（查看详情、访问链接）。

ProjectModal组件是项目详情弹窗组件，提供更详细的项目信息展示。弹窗内容包括项目大图、功能描述、技术选型、遇到的技术难点及解决方案、个人在项目中的角色和收获等。

### 5.4 问答组件

ChatWidget组件是RAG问答小部件组件，作为全局悬浮组件存在于页面上。组件默认显示为一个小按钮，点击展开问答面板。展开后显示对话区域和输入区域。组件支持最小化、关闭等操作，可以拖拽调整位置。

ChatMessage组件是问答消息展示组件，分为用户消息和AI消息两种类型。用户消息右对齐显示，AI消息左对齐显示。消息内容支持Markdown渲染，代码块提供复制按钮。AI回答的消息底部可以显示引用来源，点击可跳转至相关内容。

ChatInput组件是问答输入组件，包含文本输入框、发送按钮。输入框支持多行输入，Shift+Enter换行 Enter发送。发送前进行内容校验，空消息不允许提交。发送过程中显示加载状态，禁止重复提交。

## 六、API接口设计

### 6.1 接口规范

前后端交互采用RESTful API风格，遵循以下规范：URL使用名词复数形式表示资源，如/profile表示个人资料资源；HTTP方法表示操作类型，GET用于查询、POST用于创建；响应格式统一采用JSON，包含code、message、data三个字段；状态码遵循HTTP标准，200表示成功，400表示请求错误，500表示服务器错误。

### 6.2 问答接口

问答相关接口是RAG功能的核心：发送问答请求接口（POST /api/chat），请求体包含问题内容（question）和会话ID（sessionId，可选），返回AI生成的回答和引用来源；获取会话历史接口（GET /api/chat/history），返回指定会话或所有会话的消息记录；删除会话接口（DELETE /api/chat/sessions/:id），删除指定的会话记录。

问答接口建议使用SSE（Server-Sent Events）实现流式响应，前端需要处理SSE连接建立、消息接收、连接断开等场景。流式响应可以提供更好的用户体验，回答内容逐字显示，模拟人类打字效果。

### 6.3 个人资料接口

个人资料相关接口：获取个人基本信息接口（GET /api/profile），返回姓名、头像、职业、简介等基本信息；获取技能列表接口（GET /api/profile/skills），返回技能标签列表；获取经历列表接口（GET /api/profile/experiences），返回教育背景和工作经历列表；获取项目列表接口（GET /api/profile/projects），返回项目案例列表。

## 七、状态管理设计

### 7.1 Pinia Stores

项目使用Pinia进行状态管理，根据功能模块划分为多个store。以下是各个store的设计说明：

profileStore管理个人资料相关状态，包括基本信息、技能列表、经历列表、项目列表、加载状态等。提供获取个人资料、项目资料等actions。状态持久化不是必须的，数据通过API获取后存入store供组件使用。

chatStore管理问答相关状态，包括当前会话ID、消息历史记录、加载状态、AI响应进度等。提供发送问题、加载历史、清空会话等actions。消息历史需要持久化到本地存储，确保页面刷新后对话上下文不丢失。

themeStore管理主题相关状态，包括当前主题模式（亮色/暗黑）、主题配置信息等。提供主题切换、主题偏好保存与加载等actions。主题状态需要持久化到本地存储，用户刷新页面后保持主题偏好。

### 7.2 状态设计原则

状态管理设计遵循以下原则：首先是状态分类，将状态分为服务端数据、本地临时数据、用户偏好数据三类，不同类型采用不同的管理策略；其次是避免状态冗余，只存储必要的共享状态，组件内部状态使用ref或reactive；第三是响应式优先，优先使用计算属性（computed）派生状态，减少手动状态同步；第四是模块化组织，每个store对应一个业务模块，职责清晰，便于维护。

## 八、样式设计

### 8.1 主题系统

项目采用CSS变量实现主题系统，支持亮色和暗黑两种模式。主题变量定义在variables.scss文件中，包括颜色、字体、间距、圆角、阴影等维度。亮色主题使用浅色背景和深色文字，暗黑主题使用深色背景和浅色文字。主题切换通过修改HTML根元素的data-theme属性实现，配合CSS变量覆盖实现无缝切换。

主题变量示例如下：

```scss
// 亮色主题（默认）
:root {
```

### 8.2 全局样式

全局样式定义在src/assets/styles目录中，包括CSS重置、字体设置、全局过渡动画等。CSS重置使用normalize.css或自定义重置规则，确保各浏览器表现一致。字体设置定义全局字体栈，优先使用系统字体以提升加载性能，标题字体可以使用思源黑体等免费字体提升视觉效果。全局过渡动画应用于主题切换、元素显示隐藏等场景，使用transition属性统一控制过渡效果。

组件样式采用SCSS编写，利用SCSS的嵌套语法、变量、混合宏等功能提升样式可维护性。样式命名采用BEM命名规范，类名层次清晰，避免样式冲突。每个Vue组件的样式应设置scoped属性，实现组件级样式隔离。

### 8.3 响应式样式

响应式样式通过媒体查询实现，针对不同屏幕尺寸定义不同的样式规则。项目定义以下断点：

```scss
$breakpoint-xs: 576px;
$breakpoint-sm: 768px;
$breakpoint-md: 992px;
$breakpoint-lg: 1200px;
$breakpoint-xl: 1400px;

// 使用示例
@media (max-width: $breakpoint-sm) {
  .container {
    padding: 0 12px;
  }
}

@media (min-width: $breakpoint-lg) {
  .container {
    max-width: 1200px;
  }
}
```

## 九、安全考虑

### 9.1 XSS防护

AI回答的内容需要进行XSS过滤，防止脚本注入攻击。解决方案包括：使用DOMPurify等库对HTML内容进行净化；Vue框架默认对模板进行转义，但在使用v-html时需要确保内容已净化。

### 9.2 敏感信息保护

敏感信息如API密钥不应该存储在前端代码中，应通过环境变量管理并通过后端代理转发请求。HTTPS加密通信防止中间人攻击。

## 十、性能优化

### 10.1 首屏加载优化

首屏加载性能直接影响用户体验，优化措施包括：路由懒加载，使用defineAsyncComponent或Vue Router的动态导入实现组件按需加载；图片懒加载，使用IntersectionObserver API实现图片在进入视口时再加载；资源压缩，Vite生产构建自动压缩JS和CSS资源；CDN加速，将静态资源部署到CDN服务器；预加载关键资源，使用<link rel="preload">预加载首屏必需的资源。

### 10.2 运行性能优化

运行时性能优化措施包括：合理使用计算属性，避免不必要的计算；列表渲染使用key并启用虚拟滚动（如果项目较多）；减少DOM操作，使用v-show替代v-if控制频繁切换的元素；使用keep-alive缓存不频繁更新的组件；防抖和节流处理高频事件。

### 10.3 缓存策略

合理的缓存策略可以显著提升用户体验：浏览器缓存，静态资源设置长期缓存（hash文件名）；HTTP缓存，后端接口设置合理的Cache-Control；内存缓存，使用Pinia store缓存已获取的数据；本地存储，使用localStorage缓存不频繁变化的数据（如主题设置）。

## 十一、SEO优化

### 11.1 Meta标签

每个页面需要设置正确的Meta标签，包括title（页面标题）、description（页面描述）、keywords（关键词）等。这些信息应该包含个人姓名、技术方向等关键词，有助于搜索引擎理解页面内容。Vue项目可以使用@vueuse/head管理Meta信息。

### 11.2 结构化数据

添加JSON-LD结构化数据，帮助搜索引擎理解页面内容类型。个人主页应包含Person结构化数据，包含 name（姓名）、url（个人网站）、jobTitle（职位）、sameAs（社交链接）等字段。

### 11.3 路由模式

生产环境使用history模式（URL不带#号），并配置服务器重定向规则，将所有请求重定向到index.html，由前端路由接管。

## 十二、部署方案

### 12.1 构建配置

生产环境构建使用npm run build命令，Vite会进行以下优化：代码压缩与混淆、使用tree-shaking移除未使用代码、图片资源压缩、CSS代码提取与压缩、生成hash文件名实现缓存控制。构建产物输出到dist目录，可以部署到任何静态文件服务器。

### 12.2 部署平台

推荐部署平台包括：Vercel，提供免费额度，支持自动部署和自定义域名；Netlify，提供免费额度，支持持续部署；GitHub Pages，免费且与Git仓库无缝集成；阿里云OSS+CDN，国内访问速度快，费用合理。

### 12.3 环境配置

项目支持多环境配置：.env.development用于本地开发环境，配置开发服务器API地址；.env.production用于生产环境，配置生产服务器API地址。环境变量通过import.meta.env对象在代码中访问。
